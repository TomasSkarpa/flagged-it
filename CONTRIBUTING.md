# Contributing to Flagged-It

We welcome contributions to **Flagged-It**! By contributing, you help improve the project for everyone. Please follow the guidelines below.

## How to Contribute

1. **Fork the repository** and create a branch for your changes.  
2. **Follow the branch naming convention:**  
   - Use lowercase and kebab-case:  
     ```
     /country-guessing-game
     /dashboard-refactor
     ```
3. **Make your changes** in your branch. Keep your code clean, readable, and documented where necessary.  
4. **Commit your changes** using the commit message conventions:  
   - `feat:` for new features  
   - `fix:` for bug fixes  
   - `ref:` for refactoring  
   - `docs:` for documentation updates  
5. **Push your branch** and create a pull request. Include a clear description of your changes.  

## Code Style & Quality
 
- Write clear and concise code comments where necessary.  
- Run `make check` before submitting to ensure formatting and code analysis pass.  

## Reporting Issues

- Use GitHub Issues to report bugs or suggest features.  
- Provide as much detail as possible (steps to reproduce, screenshots, etc.).  

## Communication & Community

- Be respectful and courteous to all contributors.  
- Follow the [Code of Conduct](CODE_OF_CONDUCT.md).
